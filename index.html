DOWNLOAD THE STARTER KIT

Emacs Starter Kit for the Social Sciences

To accompany the Choosing Your Workflow Applications paper, here is a version of Phil Hagelberg's emacs starter kit with additional tools included for social scientists. The added packages include up-to-date versions of Emacs Speaks Statistics, AucTeX, and others. See the original README file for more details. If you use Mac OS X, the kit should work immediately with Emacs 23. If you use Windows or Linux, see Note 1 below for important information about what to do to get AucTeX to work.

Emacs is a very powerful editor but it is less useful out-of-the-box than it might be, in part because many convenient settings and modes are not activated by default. The starter kit is a drop-in set of nice default settings. The idea is for you to be able to download GNU Emacs, put the starter kit into ~/.emacs.d/ (or the equivalent on Windows, see below), and get to work. If you already use Emacs and have a .emacs file or ~/.emacs.d directory, the starter kit is designed to replace them, while leaving a place for you to easily append your own customizations.

The starter kit is designed to be used with GNU Emacs. It will not work with Aquamacs without some modification.

Installation Instructions (Mac OS X)

Prerequisites

0.0 Install Mac OS X's suite of Developer Tools. These came with your Mac on an install DVD of their own. Alternatively, download the most recent version from Apple directly.

0.1 Get Emacs. Download Emacs here and install it.

0.2 Install Git. Either compile it from source if you know what you're doing or install it directly.

0.3 Note your user name or the name of your computer. If you don't know either, launch emacs and do one or both of, 

C-h-v user-login-name
C-h-v system-name

You will need to know either your login name or system name to activate the final customization file properly.

Setup

1. If they're already present, back up your .emacs and .emacs.d/ directory. Then delete the .emacs file. (See below for details on where to put any customizations you wish to add yourself.)

2. Put all the files in the starter kit in ~/.emacs.d, either by unzipping it there or (if you want to do it through git) cloning the repository: 

$ git clone git://github.com/kjhealy/emacs-starter-kit ~/.emacs.d
On Windows you must copy the directory to Documents and Settings/%your user name%/Application Data/. For more details on the location of init files and directories under Windows, see the relevant sections of the Emacs for Windows FAQ.

3. Pull in some extra bits of the kit.

$ cd ~/.emacs.d/
$ git submodule init
$ git submodule update

4. Build org-mode. (The starter-kit files need this to work.)

$ cd ~/.emacs.d/src/org
$ make

4. Inside the file kjhealy.org, change the paths to any BibTeX databases as described at the top of that file.

5. Finally, change the name of the starter kit's kjhealy.org to that of %your-username%.org or %your-systemname%.org, based on the information you noted in 0.3 above. These files are where you can add in any of your own existing or future customizations without breaking the starter kit's overall setup.

6. Launch Emacs.

When you first start Emacs after installing the starter-kit, it will try to contact the ELPA repository in order to download and install some additional packages. Please be patient with it. If it doesn't get everything first time around, just launch Emacs again and it will try again.

Notes

1: Problems with AucTeX. The starter kit includes a version of AucTeX (a major Emacs mode for editing LaTeX files) that is specifically compiled for Emacs on Mac OS X. It will not work on Windows. AucTeX poses difficulties for the starter kit. It's a very useful package, so I want to include it. But unlike most Emacs lisp packages you can't run it "out of the box", so to speak. So, if you are on Windows, get the precompiled version here, install it as instructed, and then delete the vendors/auctex directory included in the starter kit. If you are on Linux, follow the instructions in the vendors/auctex/INSTALL file and recompile the package yourself. As I say, given the goals of the starter kit (to make things simpler), it's irritating that this is required.

2. I had previously included the CEDET tools and the Emacs Code Browser in the starter kit, but CEDET is presently being integrated directly into a future release of Emacs and trying to include it here makes for various compilation and configuration headaches that the starter kit is designed to avoid in the first place. So for now these packages are excluded or not activated.

Download

You can download this project in either zip or tar formats.

Or you can clone the project with Git by running: 

$ git clone git://github.com/kjhealy/emacs-starter-kit

get the source code on GitHub : kjhealy/emacs-starter-kit
